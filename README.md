# Ченнелинг-группа с Марией Шишкиной

Одностраничный лендинг камерной онлайн-встречи (ченнелинг-группы). Бренд **Внутривседома** / **Живая система**.

🌿 **Живой сайт:** https://essesum.github.io/vnutrivsedoma-site/

![Превью сайта](docs/preview-hero.png)

---

## О проекте

Статичный сайт без сборки и зависимостей — чистые **HTML + CSS + JS**.
Свёрстан по макету из Figma «Внутривседома», с коллажной эстетикой:
чёрно-белые вырезки, спокойная палитра (сирень + зелёный), заголовки **Didact Gothic**.

**Особенности:**
- Десктоп и мобильная версии (адаптив).
- Первый экран — фото-коллаж (портреты, глаз-вырезка, сиреневая «капля»).
- Плавающая шапка с постоянной кнопкой регистрации.
- Появление блоков при прокрутке + вращающийся инь-ян (отключается при `prefers-reduced-motion`).
- Коллажные вырезки в полях секций и атмосферное голографическое свечение.
- Форма заявки (имя / почта / телефон) с переходом на GetCourse.
- Доступность: подписи к полям, фокус-стили, скип-ссылка; оптимизированные изображения.

## Структура

```
index.html          — разметка страницы
css/styles.css      — стили, адаптив, анимации
js/main.js          — форма регистрации + оживление (шапка, scroll-reveal)
assets/img/         — фото и коллажные элементы
docs/               — превью для README
.github/workflows/  — авто-деплой на GitHub Pages
```

## Локальный просмотр

```bash
python3 -m http.server 8000
# открыть http://localhost:8000
```
Либо просто открыть `index.html` в браузере.

## Что настроить перед запуском

| Что | Где |
|-----|-----|
| Ссылка регистрации (GetCourse) | `REGISTRATION_URL` в начале `js/main.js` |
| Дата и стоимость участия | блок «Участие и регистрация» (`#registration`) в `index.html` |
| Отзывы | `assets/img/review1–4.png` — пример из макета, можно заменить |

Если `REGISTRATION_URL` пустой — форма показывает сообщение-заглушку.
Когда вписана ссылка — кнопка героя, шапки и форма ведут на GetCourse
(имя/почта/телефон передаются в параметрах).

## Деплой

Любой пуш в ветку `main` автоматически публикует сайт через GitHub Pages
(workflow `.github/workflows/deploy.yml`, источник — **GitHub Actions**).
