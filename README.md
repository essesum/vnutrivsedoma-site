## О проекте

Статичный лендинг **без сборки и зависимостей** — чистые HTML + CSS + JS.
Открывается в любом браузере, редактируется в любом текстовом редакторе.
Свёрстан по макету Figma «Внутривседома», эстетика — чёрно-белый коллаж,
спокойная палитра (сирень + зелёный), заголовки Didact Gothic.

**Возможности:** десктоп и мобильная версии, коллажный первый экран,
липкая шапка с кнопкой регистрации, анимации появления при прокрутке,
форма заявки (имя / почта / телефон) с переходом на GetCourse, доступность
(подписи полей, фокус-стили, поддержка «уменьшить движение»).

## Структура

```
index.html              вся разметка и тексты страницы
css/styles.css          стили, адаптив, анимации (есть :root с цветами/шрифтами)
js/main.js              форма регистрации + появление блоков при прокрутке
assets/img/             фото и коллажные элементы
docs/preview-hero.png   превью для README
```

## Локальный запуск

Можно просто открыть `index.html` в браузере. Либо поднять локальный сервер:

```bash
python3 -m http.server 8000
# открыть http://localhost:8000
```

## Как редактировать контент

**Тексты** — прямо в `index.html`, секции подписаны комментариями
(`<!-- 02 Суть формата -->` и т.д.). Меняете текст между тегами.

**Дата и стоимость участия** — блок «Участие и регистрация» (найдите
`id="registration"` в `index.html`).

**Кнопка регистрации (GetCourse)** — откройте `js/main.js`, в самом верху:

```js
const REGISTRATION_URL = ""; // вставьте ссылку GetCourse
```
- Пусто → форма показывает сообщение-заглушку.
- Указана ссылка → кнопки в шапке/герое и форма ведут на GetCourse
  (имя, почта, телефон передаются в параметрах ссылки).

**Цвета и шрифты** — вверху `css/styles.css` в блоке `:root`
(`--green`, `--lavender`, `--lilac-bg`, шрифты `--serif`/`--sans`).

**Кадр фото в первом экране на мобильной** — в `css/styles.css`,
медиазапрос `@media (max-width: 640px)`, правило `.hero__bg`
(`background-position` / `background-size`).

> ⚠️ **Кэш-версия.** В `index.html` к `css/styles.css?v=16` и
> `js/main.js?v=16` добавлен номер версии. **После любых правок CSS/JS
> увеличивайте число** (`?v=16`, `?v=17` …) — иначе у посетителей и в CDN
> может остаться старая версия из кэша.

## Деплой

Любой `git push` в ветку `main` автоматически публикует сайт через
GitHub Pages (workflow `.github/workflows/deploy.yml`).
В настройках репозитория **Settings → Pages → Source = GitHub Actions**.

Проверить, что свежая версия открылась (минуя кэш): добавьте к адресу
любую метку, напр. `…/vnutrivsedoma-site/?v16`.
