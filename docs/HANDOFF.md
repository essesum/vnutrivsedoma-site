# Инструкция по передаче проекта

Лендинг ченнелинг-группы. **Статичный сайт: HTML + CSS + JS, без сборки.**
Это НЕ npm-проект — нет `package.json`, нет бандлера, нет шага «build».
«Собрать» = просто взять файлы как есть и выложить на хостинг.

> В инструкции `ВАШ-АККАУНТ` — это GitHub-аккаунт/организация, куда вы зальёте
> проект; `ваш-поддомен.домен.рф` — ваш поддомен.

---

## 1. Для Codex / ИИ-агента, который будет дорабатывать

- **Стек:** ванильный HTML/CSS/JS. Точка входа — `index.html`,
  стили — `css/styles.css`, логика — `js/main.js`. Внешних зависимостей нет
  (только шрифты Google Fonts по CDN).
- **Не добавляйте сборку.** Никаких npm install / webpack / vite — проект
  работает открытием `index.html`. Шаг компиляции не нужен и не предусмотрен.
- **Локальный запуск:** в корне выполнить `python3 -m http.server 8000`
  и открыть `http://localhost:8000`.
- **Пути к ассетам — относительные** (`css/styles.css`, `assets/img/...`),
  поэтому сайт одинаково работает и в подпапке, и на своём домене/поддомене (в корне).
- **Кэш:** в `index.html` у `css/styles.css?v=N` и `js/main.js?v=N` есть номер
  версии. После правок CSS/JS **увеличивайте N** — иначе у пользователей
  и в CDN останется старая версия.
- **Где менять контент:**
  - тексты — в `index.html` (секции подписаны комментариями);
  - дата и стоимость — блок `id="registration"`;
  - ссылка регистрации (GetCourse) — `REGISTRATION_URL` в начале `js/main.js`;
  - цвета и шрифты — блок `:root` вверху `css/styles.css`.

## 2. Залить проект в свой GitHub и опубликовать

1. Создать новый репозиторий в `ВАШ-АККАУНТ` (напр. `vnutrivsedoma-site`).
2. Загрузить туда содержимое архива (через веб: «Add file → Upload files»,
   либо `git push`).
3. Включить хостинг: **Settings → Pages → Source = GitHub Actions**.
   В проекте уже есть воркфлоу `.github/workflows/deploy.yml` — он публикует
   сайт автоматически при каждом пуше в ветку `main`.
4. Сайт откроется по адресу `https://ВАШ-АККАУНТ.github.io/vnutrivsedoma-site/`.

> Альтернатива без GitHub: это статичные файлы, их можно залить на любой
> статический хостинг (Netlify, Vercel, обычный shared-хостинг, S3) —
> просто положить `index.html`, `css/`, `js/`, `assets/` в корень. Сборка не нужна.

## 3. Подключение вашего поддомена

Через GitHub Pages — бесплатно, с HTTPS.

1. **DNS вашего домена** → добавить одну запись:
   ```
   Тип:      CNAME
   Имя:      ваш-поддомен          (то, что слева от домена)
   Значение: ВАШ-АККАУНТ.github.io.
   ```
   (CNAME указывает на `ВАШ-АККАУНТ.github.io`, а не на полный адрес сайта.)

2. **GitHub** → репозиторий → **Settings → Pages → Custom domain** →
   ввести полный поддомен (`ваш-поддомен.домен.рф`) → **Save**.
   GitHub сам создаст в репозитории файл `CNAME`.

3. Подождать применения DNS (от минут до пары часов), дождаться галочки
   «DNS check successful».

4. В **Settings → Pages** включить **Enforce HTTPS** (после выпуска сертификата).

5. В `index.html` обновить мета-теги на новый адрес: `canonical`, `og:url`,
   `og:image`. **Относительные пути к css/js/картинкам менять НЕ нужно.**

> Апекс-домен (без поддомена, напр. `домен.рф`) тоже можно — тогда вместо CNAME
> нужны 4 A-записи: `185.199.108.153`, `185.199.109.153`, `185.199.110.153`,
> `185.199.111.153`. Для **поддомена достаточно одной CNAME-записи**.

## 4. Проверка формы заявок (обязательно перед запуском)

Сейчас форма (имя / почта / телефон) работает так:
- проверяет, что все поля заполнены и почта корректна;
- если `REGISTRATION_URL` **пустой** — показывает заглушку «Спасибо! Заявка
  отправлена» и очищает форму (**данные никуда не уходят** — режим заглушки);
- если `REGISTRATION_URL` **задан** — открывает GetCourse в новой вкладке
  и передаёт имя/почту/телефон в параметрах ссылки.

⚠️ **Важно:** сама форма НЕ сохраняет заявки. Чтобы заявки реально приходили,
нужен один из вариантов:
- **A. GetCourse-форма** — вставить готовую форму/виджет GetCourse (он собирает
  заявки сам); тогда наша форма не нужна, либо кнопка просто ведёт на GetCourse.
- **B. Сервис форм** (Formspree, Google Forms, Tilda-форма) — заменить обработчик.
- **C. Свой бэкенд** — слать POST на свой обработчик.

### Тест-чек-лист заявки

1. **Валидация (работает всегда):** пустая форма → ошибка «заполните все поля»;
   почта без `@` → ошибка про почту; корректно заполнено → принимает.
2. **С заглушкой (`REGISTRATION_URL` пуст):** после отправки — «Спасибо…»,
   поля очищаются (заявка никуда не ушла — норма для проверки вёрстки).
3. **С подключённым GetCourse:** вписать ссылку в `REGISTRATION_URL`
   (`js/main.js`), поднять `?v=N`, задеплоить; отправить тестовую заявку;
   убедиться, что открылась страница GetCourse с `?name=…&email=…&phone=…`;
   **проверить в админке GetCourse, что заявка пришла.**
4. Кнопки **«Зарегистрироваться»** в шапке и в первом экране ведут туда же, куда форма.

## 5. Что внести перед публичным запуском

- Ссылка GetCourse — `REGISTRATION_URL` в начале `js/main.js`.
