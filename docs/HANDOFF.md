# Инструкция по передаче проекта

Лендинг ченнелинг-группы. **Статичный сайт: HTML + CSS + JS, без сборки.**
Это НЕ npm-проект — нет `package.json`, нет бандлера, нет шага «build».
«Собрать» = просто взять файлы как есть и выложить на хостинг.

---

## Для Codex / ИИ-агента, который будет дорабатывать

- **Стек:** ванильный HTML/CSS/JS. Точка входа — `index.html`,
  стили — `css/styles.css`, логика — `js/main.js`. Внешних зависимостей нет
  (только шрифты Google Fonts по CDN).
- **Не добавляйте сборку.** Никаких npm install / webpack / vite — проект
  работает открытием `index.html`. Шаг компиляции не нужен и не предусмотрен.
- **Локальный запуск:** в корне выполнить `python3 -m http.server 8000`
  и открыть `http://localhost:8000`.
- **Пути к ассетам — относительные** (`css/styles.css`, `assets/img/...`),
  поэтому сайт одинаково работает и в подпапке (`…github.io/vnutrivsedoma-site/`),
  и на корневом домене.
- **Кэш:** в `index.html` у `css/styles.css?v=N` и `js/main.js?v=N` есть номер
  версии. После правок CSS/JS **увеличивайте N** — иначе у пользователей
  и в CDN останется старая версия.
- **Деплой:** `git push` в ветку `main` → GitHub Actions
  (`.github/workflows/deploy.yml`) → GitHub Pages. Файлы в `.github/workflows/`
  пушьте по SSH или правьте через веб-GitHub (у токена обычно нет `workflow`-scope).
- **Где менять контент:**
  - тексты — в `index.html` (секции подписаны комментариями);
  - дата и стоимость — блок `id="registration"`;
  - ссылка регистрации (GetCourse) — `REGISTRATION_URL` в начале `js/main.js`;
  - цвета и шрифты — блок `:root` вверху `css/styles.css`.

## Если у вас уже есть свой домен

Сайт можно повесить на ваш домен (например `vnutrivsedoma.ru`).

### Вариант 1 — оставить на GitHub Pages, привязать домен (бесплатно)

1. Репозиторий → **Settings → Pages → Custom domain** → ввести домен → **Save**
   (GitHub создаст в репозитории файл `CNAME`).
2. В DNS вашего домена добавить записи:
   - **Апекс-домен** (`vnutrivsedoma.ru`) — четыре A-записи на IP GitHub Pages:
     ```
     185.199.108.153
     185.199.109.153
     185.199.110.153
     185.199.111.153
     ```
     (по желанию AAAA для IPv6: `2606:50c0:8000::153`, `…8001::153`, `…8002::153`, `…8003::153`)
   - **Поддомен** (`www.vnutrivsedoma.ru`) — запись **CNAME** на `essesum.github.io`.
3. Подождать обновление DNS (от минут до нескольких часов), затем
   в **Settings → Pages** включить **Enforce HTTPS**.
4. В `index.html` поправить мета-теги `canonical`, `og:url`, `og:image`
   на новый домен (сейчас там адрес `essesum.github.io`).
   **Относительные пути к css/js/картинкам менять НЕ нужно** — они уже работают на корне.

### Вариант 2 — перенести на свой хостинг / другую платформу

Это статичные файлы, поэтому подойдёт любой статический хостинг
(Netlify, Vercel, обычный shared-хостинг, S3 и т.п.):
- залить `index.html` и папки `css/`, `js/`, `assets/` в корень сайта;
- сборка не требуется;
- относительные пути работают и в подпапке, и на корне.

> Tilda: перенести «как есть» в Tilda нельзя (там блочный редактор), но можно
> вставить через блок **T123 (HTML-код)** или Zero Block — содержимое
> `index.html` + подключение `css/styles.css` и `js/main.js`.
