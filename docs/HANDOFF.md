# Инструкция по передаче проекта

Лендинг ченнелинг-группы. **Статичный сайт: HTML + CSS + JS, без сборки.**
Это НЕ npm-проект — нет `package.json`, нет бандлера, нет шага «build».
«Собрать» = просто взять файлы как есть и выложить на хостинг.

---

## 1. Для Codex / ИИ-агента, который будет дорабатывать

- **Стек:** ванильный HTML/CSS/JS. Точка входа — `index.html`,
  стили — `css/styles.css`, логика — `js/main.js`. Внешних зависимостей нет
  (только шрифты Google Fonts по CDN).
- **Не добавляйте сборку.** Никаких npm install / webpack / vite — проект
  работает открытием `index.html`. Шаг компиляции не нужен и не предусмотрен.
- **Локальный запуск:** в корне выполнить `python3 -m http.server 8000`
  и открыть `http://localhost:8000`.
- **Пути к ассетам — относительные** (`css/styles.css`, `assets/img/...`),
  поэтому сайт одинаково работает и в подпапке (`…github.io/vnutrivsedoma-site/`),
  и на своём домене/поддомене (в корне).
- **Кэш:** в `index.html` у `css/styles.css?v=N` и `js/main.js?v=N` есть номер
  версии. После правок CSS/JS **увеличивайте N** — иначе у пользователей
  и в CDN останется старая версия.
- **Деплой:** `git push` в ветку `main` → GitHub Actions
  (`.github/workflows/deploy.yml`) → GitHub Pages. Файлы в `.github/workflows/`
  пушьте по SSH или правьте через веб-GitHub (у токена обычно нет `workflow`-scope).
- **Где менять контент:**
  - тексты — в `index.html` (секции подписаны комментариями);
  - дата и стоимость — блок `id="registration"`;
  - ссылка регистрации (GetCourse) — `REGISTRATION_URL` в начале `js/main.js`;
  - цвета и шрифты — блок `:root` вверху `css/styles.css`.

## 2. Подключение поддомена (основной сценарий)

Сайт вешается на ваш **поддомен**, например `zoom.livesystem.online`
или `vnutrivsedoma.livesystem.online`. Через GitHub Pages — бесплатно, с HTTPS.

**Шаги:**

1. **DNS вашего домена** → добавить одну запись:
   ```
   Тип:   CNAME
   Имя:   zoom           (поддомен, который хотите; для vnutrivsedoma.livesystem.online → "vnutrivsedoma")
   Значение: essesum.github.io.
   TTL:   по умолчанию
   ```
   (CNAME указывает именно на `essesum.github.io`, а не на полный адрес сайта.)

2. **GitHub** → репозиторий `vnutrivsedoma-site` → **Settings → Pages →
   Custom domain** → ввести полный поддомен (`zoom.livesystem.online`) → **Save**.
   GitHub сам создаст в репозитории файл `CNAME` с этим адресом.

3. Подождать применения DNS (от нескольких минут до пары часов).
   GitHub проверит домен (галочка «DNS check successful»).

4. В **Settings → Pages** включить **Enforce HTTPS** (появится, когда выпустится
   сертификат — обычно в течение часа).

5. В `index.html` обновить мета-теги на новый адрес (сейчас там `essesum.github.io`):
   `canonical`, `og:url`, `og:image`. **Относительные пути к css/js/картинкам
   менять НЕ нужно** — они уже работают в корне домена.

> Апекс-домен (без `www`, напр. `livesystem.online`) тоже можно — тогда вместо
> CNAME нужны 4 A-записи на `185.199.108.153`, `185.199.109.153`,
> `185.199.110.153`, `185.199.111.153`. Но для **поддомена достаточно одной
> CNAME-записи** (вариант выше).

## 3. Проверка формы заявок (обязательно перед запуском)

Сейчас форма (имя / почта / телефон) работает так:
- проверяет, что все поля заполнены и почта корректна;
- если `REGISTRATION_URL` **пустой** — показывает сообщение-заглушку
  «Спасибо! Заявка отправлена» и очищает форму (**данные никуда не уходят** —
  это режим заглушки);
- если `REGISTRATION_URL` **задан** — открывает GetCourse в новой вкладке
  и передаёт имя/почту/телефон в параметрах ссылки.

⚠️ **Важно:** сама по себе форма НЕ сохраняет заявки. Чтобы заявки реально
куда-то приходили, нужен один из вариантов:
- **A. GetCourse-форма** — вставить готовую форму/виджет GetCourse (он сам
  собирает заявки). Тогда наша форма не нужна, либо кнопка просто ведёт на
  страницу GetCourse.
- **B. Сервис форм** (Formspree, Google Forms, Tilda-форма и т.п.) — заменить
  `action`/обработчик на endpoint сервиса.
- **C. Свой бэкенд** — слать POST на свой обработчик.

### Тест-чек-лист заявки

1. **Валидация (работает всегда):**
   - отправить пустую форму → должна появиться ошибка «Пожалуйста, заполните все поля»;
   - ввести почту без `@` → ошибка «Проверьте, пожалуйста, адрес почты»;
   - заполнить корректно → форма принимает.
2. **С заглушкой (`REGISTRATION_URL` пуст):** после отправки видно
   «Спасибо! Заявка отправлена», поля очищаются. (Заявка никуда не ушла — норма для теста вёрстки.)
3. **С подключённым GetCourse:**
   - вписать ссылку в `REGISTRATION_URL` (`js/main.js`), поднять `?v=N`, задеплоить;
   - заполнить форму тестовыми данными и отправить;
   - убедиться, что открылась нужная страница GetCourse и в адресе есть
     `?name=…&email=…&phone=…`;
   - **проверить в админке GetCourse, что тестовая заявка пришла.**
4. Кнопки **«Зарегистрироваться»** в шапке и в первом экране ведут туда же,
   куда форма (на GetCourse, если ссылка задана).
